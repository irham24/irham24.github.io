Deepwater Horizon oil spill analysis using Synthetic-aperture radar data _____________________________________
I used the data of Envisat, a former operational satellite of European Space Agency (ESA) equipped with the SAR.
I chose to use this data because the event I illustrate in this map was in 2010, and at that time this satellite was one of the operational satellites.
At the same time, the radar data is very good for separating oil spills captured in a satellite image from the rest of the scene.
For the radar data processing part, I used ESA's SNAP software. QGIS was used to separate the Boolean band that highlighted the oil leak and the rest of the scene from the rest of the bands attached to the satellite product processed in SNAP. I also used QGIS to vectorize oil leaks. For the grinding part such as the application of morphological filters in order to reduce the surplus of pixels not representative for the event, we used the SAGA GIS software. And to make the cartographic product I used ArcGIS Pro.

Vector label in IQMaps

Engineering
×
Label a vector in IQMaps _____________________________________
I made it possible to view the labels for the vectors in IQMaps. This did not exist before.
Issue: IQMaps software does not allow to include vector label.
At my job, we need to visualize the label with the depth of the current points in order to detect certain underground utilities.
Normally this visualization would be done by switching through tabs from one software to another, but it is very expensive in terms of time, especially when these electricity points can be up to 500 in number.
The classic solution is to use an additional monitor that allows the viewing of 2 software at the same time, but which brings additional costs from a financial point of view such as the purchase of the monitor for each employee and its current consumption.
My solution allows the generation of labels for the desired vector and the import of this label in the IQMaps software, thus having the whole working environment on a single monitor.
The process consists of dividing the extension that comprises the target vector into several tiles, rasterizing these tiles, separating the background text, vectoring the text for each tile and combining them final into a single layer.
Everything is automated through a python script integrated in QGIS which also has a user interface.

SAR in human settlements footprint extraction for Timisoara Metropolitan Area - western part _____________________________________
Coherence: The regions that show stability between 2 different acquisition times have a high coherence. For example, urban areas.
Thus the opposite is given by the areas that show changes between 2 different acquisition times and are manifested in low coherence. For example, forests due to the movement of trees from the wind.
Amplitude: Urban areas and those with a volume of vegetation register a high amplitude due to the phenomena of double bounce and volume scattering.
Flat areas without texture and with certain properties of the dielectric constant such as wave-free aquatic surfaces and empty lands have a low amplitude.
RGB composite interpretation: The red colored elements are represented by surfaces with low amplitude and high coherence such as empty lands.
The yellow elements are representative of urban areas because they capture a high amplitude doubled by a high coherence.
Human settlements footprint extraction: Urban areas have a high amplitude due to the double bounce effect and the high coherence is given by the stability in time of the parts that make up the picture of the human settlement.
Based on this theory, the elements representative of human settlements were differentiated and extracted.

Clear view of vector points by categories in IQMaps _____________________________________
I have improved a work process by almost 1000%.
Issue: IQMaps software does not allow you to symbolize a vector layer by categories, nor does it allow you to set the size of a vector point to be clearly seen.
The classic solution was to select each category from the desired vector, export it, create a buffer, and repeat the steps to cover all categories. The whole process is manual and time consuming.
My solution automates the steps described above with a QGIS python script that comes with a user interface that can be used by anyone.
The classic solution takes up to 600 seconds for an 8-category vector, and my solution takes less than 60 seconds for a vector with the same number of categories. This translates into a process improvement of almost 1000%.

I was always amazed by the fidelity with which a displacement map can be made using RADAR data, reaching details of a few cm.
